# Route Planner Timeline UI Improvement Plan

## Executive Summary

**Current Score: 49/100** - Below average, needs significant improvement

The planner-prototype.html timeline view is functional but lacks critical features found in competing route planners (Google Maps, Citymapper, Roadtrippers). Major issues: redundant information, missing details (stage time, host), basic map visualization, and zero post-planning editing capabilities.

**Target Score: 75-80/100** after implementing prioritized improvements.

## New Information to Add (Priority Ordered)

### P0: Critical Missing Data
1. **Stage Time / Set Duration** ⭐⭐⭐
   - Field: `mic.stageTime` (e.g., "3-5min" or "5min set")
   - Location: Left column with time
   - Why: Essential for planning how long you'll be at each venue

2. **Host/Contact Information** ⭐⭐⭐
   - Field: `mic.host` → extract Instagram handle
   - Location: Below meta row
   - Why: Builds trust, shows personality, allows direct contact

3. **Arrival Time Calculation** ⭐⭐⭐
   - Formula: `micStartTime - transitTime - 15min buffer`
   - Display: "Arrive by 6:45 PM" (in addition to "7:00 PM start")
   - Why: Realistic planning - accounts for signup/seating

4. **Mic Name (distinct from venue)** ⭐⭐
   - Example: "The Fear City Comedy Club" (venue) vs "Fear City Mic" (event name)
   - Many venues host multiple mics - differentiate them
   - Location: Subtitle under venue name

### P1: High Value Additions
5. **Per-card "Get Directions" button** ⭐⭐⭐
   - Currently only available in modal
   - Should be on every card for quick access
   - Opens Google Maps / Apple Maps with destination

6. **Per-card "Remove from Route" button** ⭐⭐⭐
   - Zero editing capability currently
   - Allow user to remove a mic and recalculate route
   - Confirmation dialog before removing

## Post-Planning Editing Features

### Remove Mic from Route
```javascript
function removeMicFromRoute(micId) {
    if (!confirm('Remove this mic from your route?')) return;

    // Filter out mic
    currentRoute.sequence = currentRoute.sequence.filter(e => e.mic.id !== micId);

    // Recalculate transit times between remaining mics
    recalculateTransitTimes(currentRoute.sequence);

    // Re-render timeline + map
    renderResults(currentRoute.sequence, currentRoute.origin);

    showToast('Mic removed from route');
}
```

### Add Another Mic (Time Permitting)
- Show search modal
- User selects mic from filtered list
- Find optimal insertion point in sequence
- Recalculate and re-render

### Drag-to-Reorder (Optional - Lower Priority)
- Use SortableJS library
- Drag handle on each card
- Recalculate transit after reorder
- **Warning**: May produce sub-optimal route
