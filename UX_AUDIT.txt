================================================================================
                     BRUTAL UX AUDIT: planner-prototype.html
================================================================================

OVERVIEW
--------
Comprehensive UX analysis of every interaction in the Plan My Night mic crawl
planner. Each interaction graded 1-100 on 10 factors.

TARGET: 90+ on all factors before completion.

================================================================================
                           GRADING FACTORS (1-100)
================================================================================

#  | Factor            | Description
---|-------------------|-----------------------------------------------------
1  | Visual Hierarchy  | Is importance/order clear?
2  | Affordance        | Does it look clickable/interactive?
3  | Feedback          | Does it respond when interacted with?
4  | Accessibility     | Keyboard nav, focus states, ARIA labels?
5  | Touch Target      | >= 44x44px for mobile?
6  | Error Handling    | Graceful failures?
7  | Loading States    | Clear progress indication?
8  | Responsiveness    | Works on all screen sizes?
9  | Consistency       | Matches rest of UI patterns?
10 | Delight           | Animations, micro-interactions?

================================================================================
                       CRITICAL ISSUES (Must Fix First)
================================================================================

1. ACCESSIBILITY IS BROKEN (avg 43/100)
   - No ARIA labels on most interactive elements
   - Hidden radio inputs have no focus indication
   - Search dropdown lacks combobox pattern
   - Toasts lack role="alert"
   - Map is completely inaccessible

2. TOUCH TARGETS TOO SMALL (6 interactions below 44px)
   - Search clear button: 24px
   - Neighborhood chips: 26px
   - Card action buttons: ~32px
   - Show live times: no padding

3. LOADING STATES MISSING
   - Geolocation has no spinner
   - Search has no loading skeleton
   - Map has no skeleton
   - Directions button has no feedback

4. ERROR HANDLING WEAK
   - No recovery options on failures
   - No validation on time inputs
   - No specific guidance on no results

================================================================================
                          IMPLEMENTATION PHASES
================================================================================

PHASE 1: Accessibility (Critical)
---------------------------------
1. Add proper ARIA attributes to all form controls
2. Add role="alert" and aria-live to toasts
3. Add combobox pattern to search
4. Add focus-visible rings to all interactive elements
5. Add aria-labels to icon-only buttons

PHASE 2: Touch Targets
----------------------
1. Increase all buttons to minimum 44px
2. Increase clear button to 36px
3. Increase neighborhood chips to 36px
4. Add proper padding to inline buttons

PHASE 3: Loading & Feedback
---------------------------
1. Add skeleton loaders (search, map)
2. Add spinners to async buttons
3. Add progress animations
4. Add success/error feedback animations

PHASE 4: Delight & Polish
-------------------------
1. Add staggered animations for lists
2. Add micro-interactions on selection
3. Smooth accordion transitions
4. Card entrance animations

PHASE 5: Error Handling
-----------------------
1. Add validation to time inputs
2. Add recovery buttons on errors
3. Add specific "No Results" suggestions
4. Add "Try Again" flows
